Simple Shot Detector Service. Args that start with '--' (eg. -v) can also be
set in a config file (/etc/shot-detector-plot.cnf or /etc/shot-detector-
plot.ini or /etc/shot-detector-plot.conf or /etc/shot-detector-plot.yaml or
config/shot-detector-plot.cnf or config/shot-detector-plot.ini or config/shot-
detector-plot.conf or config/shot-detector-plot.yaml or config/default.cnf or
config/default.ini or config/default.conf or config/default.yaml or shot-
detector-plot.cnf or shot-detector-plot.ini or shot-detector-plot.conf or
shot-detector-plot.yaml or config.cnf or config.ini or config.conf or
config.yaml or specified via -c). The recognized syntax for setting (key,
value) pairs is based on the INI and YAML formats (e.g. key=value or
foo=TRUE). For full documentation of the differences from the standards please
refer to the ConfigArgParse documentation. If an arg is specified in more than
one place, then commandline values override config file values which override
defaults.

optional arguments:
  -h, --help
        show this help message and exit
  -c : (CONFIG_FILE), --config : (CONFIG_FILE)
        config file path
  -v, --version
        Shows the version
  --log-base : (path), [default is '/var/log/shot-detector-plot']
        Path to directory with logs. Note: this script has several log files.
  -i : (URI), --input-uri : (URI), [default is '{base}/{name}{ext}']
        Name of the video file input or path to the resource. You can use any
        string, that can be accepted by input ffmpeg-parameter. For example: -
        `udp://localhost:1234`, - `tcp://localhost:1234?listen`, -
        `http://localhost:8090/live.flv`.- `/mnt/raid/video.mp4`.The `input-
        uri` can be formed as a pattern. It is used to reduce the `input-uri`
        length.For example, you have several files in one directory. So you
        can specify directory with `input-uri-base` in the config and operate
        only with file names
  --ib : (b), --input-uri-base : (b), [default is '~/Videos']
        Value of base for input-uri. The `input-uri` can be formed as a
        pattern. It is used to reduce the `input-uri` length.For example, you
        have several files in one directory. So you can specify directory with
        `input-uri-base` in the config and operate only with file names
  --in : (n), --input-uri-name : (n), [default is 'video']
        Value of {name} for `input-uri`
  --ie : (e), --input-uri-ext : (e), [default is '.mp4']
        Value of {ext} for `input-uri`
  -f : (fmt), --format : (fmt)
        Force input format. The format is normally auto detected for input
        files so this option is not needed in most cases. Use it for hardware
        devices.
  --ff : (sec), --video-first-frame : (sec)
  --lf : (sec), --video-last-frame : (sec), [default is '60']
  --as : ({yes,no}), --as-stream : ({yes,no}), [default is 'no']
  --px : (text), --plot-xlabel : (text), [default is '$L$']
        X-label for graphic
  --py : (text), --plot-ylabel : (text), [default is '$t$']
  --pw : (cm), --plot-width : (cm), [default is '12.0']
  --ph : (cm), --plot-height : (cm), [default is '9.0']
  --pff : (font), --plot-font-family : (font), [default is 'DejaVu Sans']
  --pfs : (pt), --plot-font-size : (pt), [default is '14']
  --psf : ({pdf,png}), --plot-save-format : ({pdf,png}), [default is 'pdf']
  --psd : (path), --plot-save-dir : (path), [default is '.']
  --psn : (file-name.pdf), --plot-save-name : (file-name.pdf)
  -ps : ({yes,no}), --plot-show : ({yes,no}), [default is 'yes']
        Value of {ext} for `input-uri`
